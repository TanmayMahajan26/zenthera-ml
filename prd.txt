Antibiotic Susceptibility Testing (AST) is time-consuming—takes 24–48 hours

Personalized antibiotic selection for better patient outcomes
2. Problem Statement
Current clinical diagnostic methods rely on Antibiotic Susceptibility Testing (AST), which is performed as follows:
Patient sample is collected
Bacteria is cultured in laboratory conditions
Multiple antibiotic discs are applied to the culture
Growth inhibition zones are measured after 24–48 hours
Results are reported to clinicians
Key Issues:
Time Delay: 24–48 hours delays treatment decisions
Empirical Treatment: Doctors prescribe broad-spectrum antibiotics before results
Resistance Escalation: Overuse of broad-spectrum drugs accelerates resistance
Resource Intensive: Requires trained staff, culture media, and biological facilities

4. Methodology
4.1 Traditional AST Method
Step-by-step process:
Sample Collection: Patient sample (blood, urine, sputum) is collected
Bacterial Isolation: Sample is cultured on selective media
Antibiotic Application: Antibiotic discs are placed on bacterial lawn
Incubation: Plate is incubated at 35–37°C for 24 hours
Measurement: Clear zones around discs indicate sensitivity
Result Interpretation: Zones are compared to standardized criteria
4.2 Proposed Genomic & ML Method
Our approach follows this pipeline:
Whole Genome Sequencing (WGS): DNA is sequenced using NGS platforms
FASTA File Generation: Sequencing data is converted to FASTA format
Feature Extraction: k-mer analysis extracts numerical features
ML Model Training: Model learns patterns from labeled data
Prediction: New genome sequences are classified in seconds
Feature Extraction Details
K-mer analysis breaks down genomic sequences into fixed-length substrings. For example, with k=3:
Sequence: ATGCGTAA → {ATG, TGC, GCG, CGT, GTA, TAA}
K-mer frequency is counted across the genome, and these counts form feature vectors for ML models.

7. Feature Extraction: K-mer Analysis
What are K-mers?
K-mers are contiguous sequences of length k extracted from a genome. They represent the genomic "vocabulary" and encode local sequence information.
K-mer Extraction Process
Load genome from FASTA file
Extract all k-mers of chosen length (e.g., k=5)
Count frequency of each k-mer
Normalize counts (frequency/total k-mers)
Create feature vector for ML model
Example
Sequence: ATGCGTAAK=3: 
ATG, TGC, GCG, CGT, GTA, TAA
Feature Vector: [ATG: 1, TGC: 1, GCG: 1, CGT: 1, GTA: 1, TAA: 1] / 6 = [0.167, 0.167, ...]
8. Machine Learning Models
Models Used
Random Forest Classifier
Ensemble method combining multiple decision trees
Robust to overfitting
Provides feature importance scores
Parameters: n_estimators=100, max_depth=10
Logistic Regression
Probabilistic linear model for binary classification
Fast training and inference
Interpretable probability scores
Parameters: C=1.0, max_iter=1000
Training Process
Split data: 80% training, 20% testing
Standardize features using StandardScaler
Train model on labeled genome data
Evaluate with accuracy, precision, recall, F1-score
Select best-performing model for deployment

17. Expected Viva Questions & Answers
Q1: Is this practical for clinical use?
A: This is a proof-of-concept demonstrating how genomic data and machine learning can accelerate resistance prediction in advanced labs and research centers. While not suitable for primary healthcare due to cost and infrastructure requirements, it serves as a valuable tool for research-driven surveillance and decision support in diagnostic centers.
Q2: Why k-mers over full sequence alignment?
A: K-mers offer computational efficiency and capture local sequence patterns. They avoid the overhead of multiple sequence alignment and work well with ML algorithms.
Q3: How accurate is the model?
A: Accuracy depends on the antibiotic and dataset. Typically 80–95% for well-represented resistance patterns. Generalization to novel strains remains a challenge.
Q4: What about false positives/negatives?
A: We evaluate precision, recall, and F1-score to balance false positives and negatives. Threshold adjustment can prioritize sensitivity (catch all resistances) or specificity (minimize false alarms).
